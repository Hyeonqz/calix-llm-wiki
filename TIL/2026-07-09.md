## JPA 엔티티(하위 레이어)가 상위 레이어 enum에 의존할 수 없으면, 도메인 세터는 String을 받고 유효성 강제는 호출부에서 한다
태그: #architecture #jpa

`MerchantApplication`(apps/core = core-lib, 하위 레이어)에 정산주기 갱신 메서드를 추가하려는데, 정산주기 값 집합의 SSOT인 `SettlementCycle` enum은 `module-scheduler`(상위 레이어)에 있었다. 코어가 스케줄러 enum을 import하면 의존 방향 위반이라 불가능하다.

해결: 엔티티 메서드는 `String`을 받아 코드값(`"T+5"`/`"M+10"`)을 그대로 저장하고, **유효값 강제는 enum이 있는 상위 레이어(호출부)에서** `SettlementCycle.X.getCode()`로 넘긴다. 리터럴 문자열 직접 전달은 금지(오타·무효값 차단).

네이밍: 상태전이(`changeState*`)가 아니라 단순 필드 수정이므로 `update*` 계열(`updateSettlementCycle`)로, 기존 `updateSubMallProvider`와 결을 맞추되 `updateTime`도 갱신.

```java
// MerchantApplication (core) — String 파라미터
public void updateSettlementCycle(String settlementCycle) {
    this.settlementCycle = settlementCycle;
    this.updateTime = Instant.now();
}
// 호출부(scheduler) — enum으로 강제
application.updateSettlementCycle(SettlementCycle.M_PLUS_10.getCode());
```

## Mockito strict-stubs의 UnnecessaryStubbingException은 "기대된 예외 로그"와 구분해서 봐야 한다
태그: #testing #mockito

스킵 경로를 검증하려고 `given(mock.method()).willThrow(...)`로 일부러 예외를 던지면, SUT가 그걸 잡아 `log.error`로 남긴 **스택트레이스가 콘솔에 찍힌다 — 이건 기대된 노이즈지 테스트 실패가 아니다.** 실제 실패는 맨 끝의 `Unnecessary stubbings detected`(strict stubs)다.

`MockitoExtension` 기본은 `STRICT_STUBS`라, 선언했는데 실행 중 한 번도 호출 안 된 `given(...)`이 있으면 실패시킨다. 메시지가 `Following stubbing is unnecessary ... at Test.java:NN`으로 **정확한 줄을 지목**한다.

해결 우선순위: ① 진짜 안 쓰는 스텁 삭제 → ② 여러 테스트가 공유하는 setup이면 `lenient().when(...)` → ③ 최후로 `@MockitoSettings(strictness = LENIENT)`. 원칙은 "그 테스트가 실제 호출하는 것만 stub". `then(...).should()`(verify)는 strict-stub 대상이 아님(혼동 주의).

## List의 순서 보장은 ArrayList/LinkedList 공통 계약 — LinkedList의 실이점은 addFirst/중간삽입뿐
태그: #java

파일 Body 빌더가 `LinkedList.addLast`로 필드를 쌓는데 "순서 보장 때문에 LinkedList인가?"는 오해다. `ArrayList`도 `List`라 삽입 순서를 보장한다(순서를 안 지키는 건 HashSet/HashMap). 진짜 차이는 자료구조: `ArrayList`=배열(O(1) 인덱스 접근·캐시 지역성 좋음), `LinkedList`=연결리스트(O(1) `addFirst`/중간삽입, 인덱스 접근 O(n)).

끝에만 `addLast` 후 순회하는 패턴이면 `ArrayList`가 오히려 낫다. `LinkedList`가 기능적으로 의미 있는 곳은 `addFirst`(맨 앞 삽입)를 실제로 쓰는 파일 조립부(`merchantBatchBuilder.addFirst(fileStart)`)뿐이다.
