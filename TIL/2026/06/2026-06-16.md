## 배운 것

| # | 내용 |
|---|------|
| 1 | **IntelliJ HTTP Client 환경 파일**: `http-client.env.json`을 `.http` 파일과 같은 디렉토리에 두면, 실행 시 우측 상단 드롭다운에서 환경(local/dev 등)을 선택해 `{{host}}` 같은 변수를 자동 치환한다 |
| 2 | **민감 정보 분리**: API 키·시크릿은 `http-client.private.env.json`에 따로 두고 `.gitignore` 처리 |
| 3 | **POS 결제 플로우 (TK → RF)**: TK 커맨드가 AES key/iv를 발급하고, RF 커맨드에서 해당 key/iv로 `IntermediateInputs.Refund` JSON(`{"originalTransactionNo":"..."}`)을 AES/CBC/PKCS5Padding 암호화해 payload로 전송 |
| 4 | **HTTP Client 스크립트에서 AES 암호화**: IntelliJ 스크립트 엔진(GraalJS)은 내장 `crypto` 객체가 HMAC만 지원 → `Java.type('javax.crypto.Cipher')` 방식 또는 순수 JS AES-CBC 구현으로 해결 가능 |
| 5 | **response handler 전역 변수**: `client.global.set(key, value)` / `client.global.get(key)` 로 요청 간 데이터를 전달, pre-script에서는 `request.variables.set(key, value)` 로 해당 요청의 `{{변수}}`에 주입 |

## 이슈 / 트러블슈팅

| # | 이슈 | 해결 |
|---|------|------|
| 1 | Swagger에서 RF 테스트 시 payload 암호화가 불편 | `.http` 파일의 pre-request 스크립트에서 TK 응답 key/iv를 받아 자동 암호화 후 요청 전송 |
| 2 | IntelliJ 내장 `crypto` 객체가 AES를 지원하지 않음 | 순수 JS AES-CBC + PKCS7 패딩 구현을 pre-script에 직접 삽입 |
