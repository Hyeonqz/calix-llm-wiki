## 배운 것

| # | 내용 |
|---|------|
| 1 | `jackson-datatype-hibernate5`는 `javax.persistence.Transient`를 참조하는데, Spring Boot 3.x는 `jakarta.persistence.*` 네임스페이스를 사용하므로 두 버전이 충돌한다 |
| 2 | Jackson이 POJO를 직렬화할 때 등록된 `HibernateAnnotationIntrospector`가 모든 필드를 검사하므로, 호환되지 않는 `javax.persistence` 클래스를 로드하려다 실패한다 |
| 3 | 순수 JPQL 쿼리(`updatePayment`)는 Jackson이 개입하지 않아 에러가 발생하지 않지만, JSON 컬럼 직렬화(`savePaymentResult`의 `resultBody`)가 트리거 지점이 된다 |
| 4 | Spring Boot 3.x 프로젝트에서는 반드시 `jackson-datatype-hibernate6`을 사용해야 한다 |
| 5 | MDC 관리 모범사례: try-finally + `MDC.remove()` 보다 `MDC.putCloseable()` + try-with-resources가 안전하다 — 예외 발생 시에도 자동 cleanup 보장 |
| 6 | 횡단 관심사(MDC, 로깅)는 템플릿 메서드 패턴의 상위 클래스에서 처리해야 하위 구현체가 비즈니스 로직에만 집중할 수 있다 |
| 7 | `RestClientInterceptor`에서 MDC 값이 없을 때 `UNIONPAY`로 fallback하면 로그 데이터가 오염된다 — `UNKNOWN` enum 값으로 안전하게 처리해야 한다 |
| 8 | `findLatestBusinessDay()`는 "오늘 날짜" 개념이 없다 — 미래 날짜를 넘겨도 DB에 공휴일이 등록되어 있으면 동일하게 역산하여 직전 영업일을 반환한다 |
| 9 | 공휴일이 토요일과 겹치는 경우 DB에는 공휴일 단건만 insert — `holiday_date`에 unique 제약이 있으므로 중복 insert 시 오류 발생 |
