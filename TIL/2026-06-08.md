## 배운 것

| # | 내용 |
|---|------|
| 1 | **BC(Bounded Context)** — "같은 용어가 같은 의미를 갖는 경계 영역". 패키지 최상위를 BC로 분리하면 신규 클래스 위치 판단 기준이 생김 |
| 2 | **DDD vs BC 기반 차이** — DDD는 전략적(BC 분리) + 전술적(Aggregate·Domain Event) 설계를 모두 포함. BC 기반은 전략적 설계만 적용한 실용적 절충 |
| 3 | **Lite Hexagonal** — Port 인터페이스 완전 분리 없이 Inbound/Outbound Adapter 위치만 `infrastructure/kafka/inbound`, `infrastructure/kafka/outbound`로 명시하면 아키텍처 의도가 충분히 전달됨 |
| 4 | **xlsx 파일 내부 구조** — xlsx는 ZIP 파일로, `xl/worksheets/sheet1.xml` (데이터), `xl/sharedStrings.xml` (문자열 풀)을 직접 파싱 가능 |
| 5 | **헤더 이름 기반 컬럼 탐지** — 열 문자(A, B, C)를 하드코딩하면 파일 포맷 변경마다 코드 수정 필요. 헤더 행에서 이름으로 열을 탐지하면 위치 변경에 자동 대응 |
| 6 | `LocalDate`는 날짜만 보유 — `yyyyMMddHHmmss` 같은 시간 필드 포함 포매터로 `format()` 호출 시 `UnsupportedTemporalTypeException: Unsupported field: HourOfDay` 발생. 시간 정보가 필요하면 `LocalDateTime`으로 선언해야 함 |
| 7 | `BASIC_ISO_DATE` vs `ofPattern("yyyyMMdd")` — 출력은 동일(`20240101`)하지만 차이가 있음: `BASIC_ISO_DATE`는 타임존 오프셋 포함 가능하고 `.withZone()` 설정 불가. `Instant` 포맷이 필요하다면 `ofPattern` + `.withZone(ZoneId.systemDefault())` 조합 필요 |
| 8 | **응답 직렬화 패턴 추출** — `if (IS_FIXED_LENGTH) parser.encode() else += NEWLINE` 같은 3줄 패턴이 5개 커맨드 핸들러에 복사되어 있을 때, `writeResponse(ctx, holder)` 헬퍼 하나로 추출하면 변경점이 1곳으로 집약됨 |
| 9 | **전략 패턴 불일치의 위험** — `RealTimePushListener`는 `List<TcpPushHandler>` 전략 패턴, `PaymentResultCallbackListener`는 IMTaxi 로직을 인라인 처리. 동일한 Push 흐름인데 구현 방식이 달라 하나를 수정하면 다른 쪽을 놓칠 수 있음. 두 리스너를 같은 전략 패턴으로 통일해야 함 |
| 10 | **`@Transactional` on Netty IO 스레드** — Netty `channelRead0()`에 `@Transactional`을 붙이면 EventLoop 스레드가 DB 완료까지 블로킹됨. 같은 EventLoop에 묶인 다른 채널 이벤트가 대기 → 처리량 저하. 비즈니스 로직을 `@Async` 또는 별도 스레드풀로 위임해야 함 |
| 11 | **매직넘버 불일치 버그** — `TCPHandler`는 `@Value("${qrbank.tcp-qr-valid-time}")`로 타이머를 프로퍼티 관리하는데, `PaymentResultCallbackListener`는 `66`을 하드코딩. 같은 값인데 관리 방식이 달라 프로퍼티를 바꿔도 한 쪽은 변경 안 됨. 상수 혹은 프로퍼티 단일화 필요 |
| 12 | **`ChannelService.getChannel()` O(n) 조회** — `ConcurrentMap<ChannelId, ChannelInformation>`을 전체 순회하며 `transactionNo` 매칭. 보조 인덱스 `ConcurrentMap<String, ChannelId> transactionIndex`를 추가하면 O(1) 조회로 개선 가능. `setTransactionNo()` 시점에 두 맵을 동시 갱신하면 됨 |
