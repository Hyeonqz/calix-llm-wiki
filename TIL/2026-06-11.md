## 배운 것

| # | 내용 |
|---|------|
| 1 | Java Stream `.map()`은 lazy — `.collect()`, `.forEach()` 같은 터미널 연산 없이는 파이프라인이 실행되지 않는다 |
| 2 | `.map()`은 `T → R` 변환 전용, 외부 Map에 side effect를 줄 때는 `.forEach()` 를 써야 한다 |
| 3 | `cancelMap.remove(transactionNo)`를 Case1 for-loop 안에서 호출하면, 루프 종료 후 `cancelMap`에는 계좌결제 CANCEL-only 건만 남아 Case2의 stream filter가 불필요해진다 |
| 4 | `SettlementBatchHistory.getPayment()`는 CANCEL/REFUND 행이어도 원거래 PAY Payment 엔티티를 반환 → 별도 DB 조회(`findOriginalPaymentsByTransactionNos`) 없이 `isSameDayCancel()` 직접 호출 가능 |
