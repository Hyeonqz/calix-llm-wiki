## 배운 것
- **Spotless + Naver formatter 적용 방법**: IntelliJ formatter XML은 Spotless에서 지원하지 않음. Eclipse formatter XML만 사용 가능 (`eclipse().configFile(...)`)
  - Naver hackday-conventions-java 레포에서 `naver-eclipse-formatter.xml`과 `naver-intellij-formatter.xml` 두 가지를 제공함
  - IDE는 IntelliJ formatter, CI/빌드는 Eclipse formatter로 역할 분리
- **Gradle 멀티모듈 플러그인 패턴**: 루트에서 `apply false`로 버전만 선언하고, `subprojects` 블록에서 실제 적용하는 것이 표준 패턴
- **AI 에이전트(Claude Code / Codex) 포맷팅 전략**: 에이전트 지시(`CLAUDE.md`, `CODEX.md`)보다 Spotless 같은 도구 강제가 신뢰도 높음
- **IntelliJ HTTP Client JavaScript 런타임**: Node.js가 아님. `require()` 사용 불가
  - `crypto.hmac.sha256().withTextSecret(key).updateWithText(msg).digest().toBase64()` — 내장 crypto API
  - `$random.uuid()`는 스크립트 블록 안에서 동작하지 않음 → `Date.now().toString(36) + Math.random().toString(36)` 조합으로 대체

## 이슈 / 트러블슈팅
- **WSL I/O 에러**: `./gradlew spotlessApply` 실행 시 `ChecksumService` I/O 에러 발생. WSL에서 Windows 파일시스템(`/mnt/c/`) 접근 시 간헐적으로 발생하는 문제. Gradle 캐시 정리로도 해결 안 됨 → Windows PowerShell에서 `.\gradlew.bat spotlessApply` 실행으로 우회
- **HTTP Client `require is not defined`**: IntelliJ HTTP Client는 자체 JS 런타임 사용. Node.js `require('crypto')` → IntelliJ 내장 `crypto` 전역 객체로 교체하여 해결
