## 다음 후보 (추천순)

### 🥇 1순위 — 시간과 날짜 (`time-and-date`)
**왜 1순위:** encoding의 자매편. "당연해 보이는 데이터(글자/시간)가 왜 조용히 깨지나"라는 같은 결.
역사(달력)가 풍부해 "역사 → 원리" 아크가 살고, 실무 버그를 파이썬 `datetime`/`zoneinfo`로 **코드 실측**하기 딱 좋다.

- **역사:** 해시계 → 그레고리력 → 표준시/GMT → UTC → 유닉스 타임스탬프 → NTP
- **원리:** 타임존, DST(서머타임), 윤초, epoch, ISO 8601, 단조시계(monotonic) vs 벽시계(wall clock)
- **실무:** "서버는 UTC, 표시만 로컬", 2038년 문제, DST 경계 버그, `Instant` vs `LocalDateTime`, DB 타임존 함정
- **훅 후보:** "새벽 2시에 두 번 실행된 배치" / "하루 예약이 25시간이 된 날"

### 🥈 2순위 — HTTP와 웹의 기초 (`http`)
**왜:** 백엔드 주니어가 매일 쓰는데 뼈대는 흐릿한 영역.

- **역사:** 팀 버너스리 → HTTP/0.9~1.1 → HTTPS → HTTP/2 → HTTP/3(QUIC)
- **원리:** 요청/응답, 메서드·상태코드의 진짜 의미, 헤더, 쿠키·세션, 캐시, CORS, keep-alive
- **실무:** 멱등성, `301 vs 302`, 캐시 무효화, CORS 프리플라이트 지옥, 커넥션 풀
- **훅 후보:** "왜 새로고침하면 결제가 두 번 될까"

### 🥉 3순위 — 인증과 인가 (`auth`)
**왜:** "복붙으로 돌아가는데 왜 그런지는 모르는" 넘버원 주제.

- **역사:** 비밀번호 → 세션/쿠키 → 토큰 → OAuth → JWT/OIDC
- **원리:** 인증(authn) vs 인가(authz), 해싱·솔트, 세션 vs 토큰, JWT 구조·서명, refresh 토큰
- **실무:** JWT를 세션 대체로 쓰면 안 되는 이유, 토큰 저장 위치·탈취, OAuth 플로우 실전
- **훅 후보:** "로그아웃했는데 토큰이 아직 살아있다"

## 그 외 후보 풀 (아이디어 저장소)

정하기 전 참고용. 각각 "역사 → 원리 → 실무"로 충분히 풀린다.

- **TCP/IP와 네트워크** — 패킷·핸드셰이크·혼잡제어, "왜 연결이 끊겼나"
- **데이터베이스 인덱스와 쿼리 실행** — B-tree·실행계획·`EXPLAIN`, "인덱스 걸었는데 왜 안 타나"
- **Git 내부구조** — 객체·DAG·브랜치의 실체, "리베이스가 대체 뭘 하는가"
- **부동소수점과 수 표현** — `0.1 + 0.2 != 0.3`, IEEE 754, 돈 계산에 float 쓰면 안 되는 이유
- **동시성과 락** — 스레드·경쟁조건·데드락 (transaction 책과 일부 겹침 주의)
- **정규식과 문자열 처리** — 오토마타·백트래킹, "정규식이 서버를 멈춘 날"(ReDoS)
- **캐시** — 로컬~분산, 무효화·일관성, "캐시 때문에 안 바뀐 화면"
- **메모리와 GC** — 스택/힙·가비지 컬렉션, "메모리 누수는 어디서 오나"

## 고르는 기준 (참고)

1. **encoding·transaction과 결이 비슷한가** — 무심한 데이터/약속이 조용히 깨지는 주제일수록 이 책 시리즈와 잘 맞음.
2. **역사가 있는가** — "역사 → 원리" 아크가 살려면 인물·연대·일화가 필요.
3. **코드로 실증되는가** — 장마다 파이썬/JS로 버그를 재현·복구할 수 있어야 사이클(코드 실증)이 산다.
4. **주니어가 매일 쓰는데 원리는 모르는가** — 실무 가치가 크고 훅이 강함.

## 착수 방법

책을 정하면 encoding과 동일하게:
1. `content/books/{book}/` 생성 + `index.md`(스펙)·`_meta.js`(목차)·장 스텁 스캐폴딩
2. `content/books/{book}/book.json` 작성 (voice·structure·styleGuide·promptQueue·editorAgent·spec·buildCmd·commit)
3. "N장 써줘"로 `write-chapter` 사이클 시작
